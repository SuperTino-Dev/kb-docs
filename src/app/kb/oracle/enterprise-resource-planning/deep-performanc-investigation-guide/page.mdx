# Oracle EBR Deep Performance Investigation Guide

## Overview

This guide is intended for advanced Oracle performance investigations after enabling Edition-Based Redefinition (EBR).

The goal is to identify:

* SQL regressions
* Optimizer behavior changes
* Cursor invalidations
* Library Cache contention
* VPD side effects
* Editioning View overhead
* ERP Cloud EBR-related issues
* GoldenGate replication impact

---

# Investigation Workflow

Recommended order:

1. ASH Analysis
2. AWR Comparison
3. SQL Monitor Analysis
4. Cursor Invalidations
5. Child Cursor Investigation
6. Library Cache Analysis
7. Editioning View Analysis
8. VPD Analysis
9. ORA_SEED_SET Analysis
10. ERP Cloud Validation
11. GoldenGate Validation

---

# Section 1 - ASH Analysis

## Objective

Determine where sessions spend time after EBR activation.

---

### Top Wait Events

```sql
SELECT
    event,
    COUNT(*) samples
FROM v$active_session_history
GROUP BY event
ORDER BY samples DESC;
```

---

### Top SQL IDs

```sql
SELECT
    sql_id,
    COUNT(*) samples
FROM v$active_session_history
GROUP BY sql_id
ORDER BY samples DESC;
```

---

### Waits By Module

```sql
SELECT
    module,
    event,
    COUNT(*)
FROM v$active_session_history
GROUP BY module,event
ORDER BY COUNT(*) DESC;
```

---

### What To Look For

Potential EBR indicators:

* library cache lock
* library cache pin
* cursor: pin S wait on X
* row cache lock
* hard parse waits
* latch contention

# Section 3 - SQL Monitor Analysis

## Objective

Identify plan regressions in long-running SQL.

---

### Active SQL Monitor

```sql
SELECT
    sql_id,
    status,
    elapsed_time,
    cpu_time,
    buffer_gets
FROM v$sql_monitor;
```

---

### Detailed Report

```sql
SELECT DBMS_SQLTUNE.report_sql_monitor(
    sql_id => '<SQL_ID>',
    type => 'TEXT',
    report_level => 'ALL'
)
FROM dual;
```

---

### Validation

Review:

* actual rows
* estimated rows
* index usage
* join methods
* full table scans

# Section 4 - Cursor Invalidations

## Objective

Determine whether EBR deployment invalidated SQL cursors.

---

### Invalidations

```sql
SELECT
    sql_id,
    invalidations,
    executions
FROM v$sql
WHERE invalidations > 0
ORDER BY invalidations DESC;
```

---

### High Parse Activity

```sql
SELECT
    name,
    value
FROM v$sysstat
WHERE name IN (
    'parse count (hard)',
    'parse count (total)'
);
```

---

### Validation

Investigate:

* excessive hard parsing
* invalid object recompilation
* shared pool pressure

---

# Section 5 - Child Cursor Explosion

## Objective

Identify SQL statements generating excessive child cursors.

---

### Child Cursor Count

```sql
SELECT
    sql_id,
    COUNT(*) child_count
FROM v$sql
GROUP BY sql_id
HAVING COUNT(*) > 20
ORDER BY child_count DESC;
```

---

### Child Cursor Reasons

```sql
SELECT
    sql_id,
    child_number,
    reason
FROM v$sql_shared_cursor
WHERE sql_id = '<SQL_ID>';
```

---

### Common Findings

* optimizer mismatch
* bind mismatch
* statistics mismatch
* edition mismatch

---

# Section 6 - Library Cache Analysis

## Objective

Identify contention caused by EBR object changes.

---

### Library Cache Statistics

```sql
SELECT
    namespace,

    pins,
    pinhits,

```

---

### Library Cache Waits

```sql
SELECT
    event,
    total_waits,
    time_waited
FROM v$system_event
WHERE event LIKE 'library cache%';
```

---

### Validation

Look for:

* library cache lock
* library cache pin
* excessive reloads


# Section 9 - ORA_SEED_SET1 / ORA_SEED_SET2 Investigation

## Objective

Validate ERP Cloud seed-set filtering impact.

---

### Tables Using Seed Sets

```sql
SELECT
    owner,
    table_name,
    column_name
FROM dba_tab_columns
WHERE column_name IN (
'ORA_SEED_SET1',
'ORA_SEED_SET2'
);
```

---

### Distribution Analysis

```sql
SELECT
    ORA_SEED_SET1,
    COUNT(*)
FROM <TABLE_NAME>
GROUP BY ORA_SEED_SET1;
```

---

### Histogram Validation

```sql
SELECT
    owner,
    table_name,
    column_name,
    histogram
FROM dba_tab_col_statistics
WHERE column_name='ORA_SEED_SET1';
```

---

### Validation

Review:

* skewed distributions
* histogram accuracy
* filter selectivity

# Final Root Cause Decision Tree
```
If waits are:

| Wait Type               | Likely Cause               |
| ----------------------- | -------------------------- |
| library cache lock      | EBR object invalidation    |
| cursor pin S wait on X  | child cursor explosion     |
| db file sequential read | index access regression    |
| db file scattered read  | full table scan regression |
| row cache lock          | metadata contention        |
| enq: TX                 | application locking        |
| CPU                     | optimizer plan regression  |

```
The most common EBR-related root causes are:

1. Plan regressions
2. Statistics changes
3. Child cursor proliferation
4. Library cache contention
5. VPD predicate expansion
6. ORA_SEED_SET selectivity issues
7. Crossedition trigger overhead
8. Editioning View access path changes
