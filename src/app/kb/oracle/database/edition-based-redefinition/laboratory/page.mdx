# Oracle Edition-Based Redefinition (EBR) Lab

## What is EBR?

Edition-Based Redefinition (EBR) is an Oracle Database feature that allows multiple versions of application code to exist simultaneously.

This enables:

- Online upgrades
- Near zero downtime deployments
- Rolling application changes
- Safe PL/SQL migrations

Very common in:

- Oracle Fusion ERP
- Oracle E-Business Suite Online Patching
- SaaS applications

---

# Core Concept

```text
Same Data
Different Code
No Downtime

Tables are shared.
PL/SQL logic is versioned.
```

## Architecture Overview

```text
                 +----------------+
                 |   APP USERS    |
                 +--------+-------+
                          |
                    CURRENT EDITION
                          |
         -----------------------------------
         |                                 |
+--------v--------+             +----------v--------+
| EDITION V1      |             | EDITION V2        |
| pkg_orders old  |             | pkg_orders new    |
| view old        |             | view new          |
+-----------------+             +-------------------+
```

# Step 0 - Validate Database Environment

Before creating the EBR user, validate the database version, current container, database properties, and important parameters.

## Validate Database Version

```sql
SELECT banner_full
FROM v$version;
```

## Validate Current Container

```sql
SHOW CON_NAME;

SELECT
    con_id,
    name,
    open_mode,
    restricted
FROM v$pdbs
ORDER BY con_id;
```

## Validate Database Properties

```sql
SELECT *
FROM database_properties
ORDER BY property_name;
```

## Filtered Database Properties

```sql
SELECT
    property_name,
    property_value
FROM database_properties
WHERE property_name LIKE '%EDITION%'
   OR property_name LIKE '%DEFAULT%'
   OR property_name LIKE '%PDB%'
ORDER BY property_name;
```

## Validate Important Parameters

```sql
SELECT
    name,
    value,
    isdefault,
    isses_modifiable,
    issys_modifiable
FROM v$parameter
WHERE name IN (
    'compatible',
    'enable_ddl_logging',
    'plsql_code_type',
    'plsql_optimize_level'
)
ORDER BY name;
```

## Validate Current Edition

```sql
SELECT SYS_CONTEXT('USERENV','CURRENT_EDITION_NAME') AS current_edition
FROM dual;
```

Expected:

```text

```

## Validate Existing Editions

```sql
SELECT
    edition_name,
    parent_edition_name,
    usable
FROM dba_editions
ORDER BY edition_name;
```

# Step 1 - Create EBR User

```sql
CREATE USER ebr IDENTIFIED BY ebr;

GRANT CONNECT, RESOURCE TO ebr;

GRANT CREATE SESSION TO ebr;
GRANT CREATE TABLE TO ebr;
GRANT CREATE VIEW TO ebr;
GRANT CREATE PROCEDURE TO ebr;
GRANT CREATE SEQUENCE TO ebr;
GRANT CREATE SYNONYM TO ebr;

GRANT CREATE ANY EDITION TO ebr;
GRANT DROP ANY EDITION TO ebr;

ALTER USER ebr ENABLE EDITIONS;
```

## Verify User

```sql
SELECT
    username,
    editions_enabled,
    account_status,
    default_tablespace,
    temporary_tablespace
FROM dba_users
WHERE username='EBR';
```

Expected:

```text
EDITIONS_ENABLED = Y
```

# Step 4 - Create Initial Package

## Package Specification

```sql
CREATE OR REPLACE PACKAGE pkg_customers AS

    PROCEDURE show_customer(p_id NUMBER);

END;
/
```

## Package Body

```sql
CREATE OR REPLACE PACKAGE BODY pkg_customers AS

PROCEDURE show_customer(p_id NUMBER) IS

    v_name VARCHAR2(100);

BEGIN

    SELECT name
    INTO v_name
    FROM customers
    WHERE customer_id = p_id;

    DBMS_OUTPUT.PUT_LINE('V1 CUSTOMER='||v_name);

END;

END;
/
```

## Validate Package Objects

```sql
SELECT
    object_name,
    object_type,
    status,
    edition_name
FROM user_objects_ae
WHERE object_name = 'PKG_CUSTOMERS'
ORDER BY object_type;
```

# Step 10 - Validate Isolation

## Session 1

```sql
ALTER SESSION SET EDITION = ora$base;

EXEC pkg_customers.show_customer(1);
```

Expected:

```text
V1 CUSTOMER=JOHN
```

## Session 2

```sql
ALTER SESSION SET EDITION = version2;

EXEC pkg_customers.show_customer(1);
```

Expected:

```text
V2 CUSTOMER=JOHN
```

## Important Observation

```text
Both editions use the same physical table:

customers

But execute different code versions.
```

# Editioning Views

## Why Editioning Views Exist

```text
Tables are not editionable.

Oracle uses Editioning Views to abstract table structure changes.
```


# GoldenGate + EBR Considerations

GoldenGate replicates:

```text
Physical tables
```

GoldenGate does NOT automatically understand:

```text
Edition context
Editioning logic
```

## Common Issues

```text
DDL out of order
Wrong edition execution
Editioning view mismatches
Invalid synonyms
ORA-38802
ORA-38818
ORA-42314
```

## Useful GoldenGate Queries

```sql
SELECT
    owner,
    object_name,
    object_type,
    edition_name,
    status
FROM dba_objects_ae
WHERE owner = 'EBR'
ORDER BY object_name,
         edition_name;
```
